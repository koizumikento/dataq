# dataq v0.1.14 Release Notes

Released: 2026-02-27

## User-visible changes

- MCP `tools/list` now returns stricter per-tool `inputSchema` definitions, including explicit `required`, `oneOf`, enum/pattern constraints, and `additionalProperties: false`.
- MCP `tools/call` now rejects unknown arguments consistently with machine-readable `input_usage_error` payloads.
- MCP responses now expose alias deprecation warnings in `structuredContent.meta.warnings` when deprecated aliases are used.
- `dataq.recipe.replay` tool metadata now documents exit code `2` semantics for strict lock mismatch and step-level validation mismatch.
- `ingest.api` method validation now accepts HTTP methods case-insensitively (`GET`/`get`, etc.).

## Bug fixes and contract hardening

- Canonicalized MCP `invalid_params` names in error payloads for gate/assert/tool-specific argument failures.
- Added deterministic MCP coverage for schema strictness, alias warnings, unknown argument rejection, and canonical invalid parameter naming.

## Breaking changes

- None.
